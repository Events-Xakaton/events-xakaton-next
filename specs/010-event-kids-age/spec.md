# Feature Specification: Kids Audience Marker for Events

**Feature Branch**: `010-event-kids-age`  
**Created**: 2026-03-15  
**Status**: Draft  
**Input**: User description: "Добавить на ивенты инфу о том, что там можно с детьми и от какого возраста. Поле должно быть при создании, редактировании и просмотре. На карточках ивентов писать, что это ивент для детей, если это так. 90% ивентов — для взрослых."

## Clarifications

### Session 2026-03-15

- Q: Что лучше: только галочка или возраст `N+`? → A: Комбинированная модель: флаг `для детей` + возраст `N+` как опциональное уточнение.
- Q: Что должно быть дефолтом, учитывая что 90% ивентов для взрослых? → A: По умолчанию все новые и существующие ивенты считаются `не для детей`.
- Q: Какой текст показывать на карточках? → A: Бейдж `Для детей`, а если указан возраст — `Для детей {N}+`.
- Q: Где располагать метку на карточках? → A: На всех карточках метка размещается под чипом времени (в верхнем левом блоке контента).

## User Scenarios & Testing _(mandatory)_

### User Story 1 - Создание ивента с признаком детской аудитории (Priority: P1)

Как организатор, я могу при создании отметить, что ивент для детей, и при необходимости указать возраст `N+`, чтобы пользователи сразу понимали целевую аудиторию.

**Why this priority**: это основной вход для новых данных и главная бизнес-ценность фичи.

**Independent Test**: создать два ивента (обычный и детский `6+`) и проверить, что оба корректно сохраняются и отображаются.

**Acceptance Scenarios**:

1. **Given** форма создания ивента открыта, **When** организатор не включает признак `Для детей`, **Then** ивент создается как взрослый (без детского бейджа).
2. **Given** форма создания ивента открыта, **When** организатор включает `Для детей` и задает `6+`, **Then** ивент сохраняется как детский с возрастным порогом `6`.
3. **Given** форма создания ивента открыта, **When** организатор включает `Для детей`, но не задает возраст, **Then** ивент сохраняется как детский без уточнения возраста.

---

### User Story 2 - Редактирование детских параметров (Priority: P1)

Как организатор, я могу изменить признак `Для детей` и возраст `N+` в уже созданном ивенте, чтобы поддерживать актуальность информации.

**Why this priority**: без редактирования данные быстро устаревают и теряют ценность.

**Independent Test**: открыть существующий ивент в режиме редактирования и выполнить переходы `взрослый -> детский 6+ -> детский без возраста -> взрослый`.

**Acceptance Scenarios**:

1. **Given** ивент взрослый, **When** организатор включает `Для детей` и сохраняет, **Then** после сохранения ивент отображается как детский.
2. **Given** детский ивент с возрастом `6+`, **When** организатор меняет возраст на `12+`, **Then** на всех экранах показывается `12+`.
3. **Given** детский ивент, **When** организатор выключает `Для детей`, **Then** возраст очищается и детские бейджи исчезают.

---

### User Story 3 - Просмотр ивента и распознавание на карточках (Priority: P1)

Как пользователь, я вижу в карточках и деталях, что ивент детский (и с какого возраста, если указано), чтобы быстрее принимать решение об участии.

**Why this priority**: именно это закрывает пользовательскую задачу "понять, можно ли с детьми".

**Independent Test**: проверить отображение на карточках ленты/профиля/клуба и на экране деталей для трех вариантов: взрослый, детский без возраста, детский с возрастом.

**Acceptance Scenarios**:

1. **Given** ивент отмечен как детский без возраста, **When** карточка отображается в списке, **Then** показывается бейдж `Для детей`.
2. **Given** ивент отмечен как детский с возрастом `6+`, **When** карточка отображается в списке, **Then** показывается бейдж `Для детей 6+`.
3. **Given** ивент отмечен как детский, **When** карточка отображается в ленте/профиле/клубе, **Then** бейдж расположен под чипом времени.
4. **Given** ивент не отмечен как детский, **When** карточка или детали открыты, **Then** детские метки не показываются.

---

### Edge Cases

- Существующие ивенты, созданные до фичи, должны интерпретироваться как взрослые без ошибок UI.
- Переключение `Для детей` в `false` в режиме редактирования должно сбрасывать ранее выбранный возраст.
- Длинный текст бейджа (например, локализация) не должен ломать верстку карточек.
- Неконсистентные данные от API (`isForKids=false`, `kidsMinAge!=null`) должны отображаться как взрослый ивент.

## Requirements _(mandatory)_

### Functional Requirements

- **FR-001**: Система MUST поддерживать в модели ивента признак `isForKids: boolean`.
- **FR-002**: Система MUST поддерживать в модели ивента поле `kidsMinAge: number | null` (возраст `N+`, если задан).
- **FR-003**: Для create-flow система MUST по умолчанию выставлять `isForKids=false` и скрывать поле возраста, пока признак не включен.
- **FR-004**: В create-flow система MUST позволять сохранить детский ивент как с указанным возрастом `N+`, так и без него.
- **FR-005**: В edit-flow система MUST позволять включать/выключать признак `Для детей` и изменять возраст `N+`.
- **FR-006**: При выключении признака `Для детей` система MUST очищать `kidsMinAge` перед сохранением.
- **FR-007**: На экране просмотра ивента система MUST показывать признак детской аудитории и возраст `N+`, если возраст указан.
- **FR-008**: На карточках ивентов (в ленте, профиле и списках клуба) система MUST показывать бейдж `Для детей` или `Для детей {N}+` для детских ивентов.
- **FR-009**: Для взрослых ивентов система MUST не показывать детские бейджи и блоки.
- **FR-010**: Система MUST корректно обрабатывать старые ивенты без новых полей (backward compatibility без визуальных/runtime ошибок).
- **FR-011**: Система MUST использовать единый текстовый формат `Для детей` и `Для детей {N}+` на всех карточках и в деталях.
- **FR-012**: На карточках ленты/профиля/клуба система MUST размещать детский бейдж под чипом времени в верхнем левом блоке карточки.

### Key Entities _(include if feature involves data)_

- **EventAudience**: атрибуты целевой аудитории ивента: `isForKids`, `kidsMinAge`.
- **EventAudienceBadge**: вычисляемый текстовый лейбл для UI-карточек: `none | Для детей | Для детей {N}+`.

## Success Criteria _(mandatory)_

### Measurable Outcomes

- **SC-001**: В 100% smoke-сценариев create/edit/view данные `isForKids/kidsMinAge` сохраняются и читаются без расхождений.
- **SC-002**: В 100% карточек детских ивентов отображается корректный бейдж (`Для детей` или `Для детей {N}+`).
- **SC-003**: Для 100% взрослых ивентов (включая legacy записи) детский бейдж отсутствует.
- **SC-004**: После выключения признака `Для детей` в edit-flow возраст `N+` не отображается ни в деталях, ни в карточках.
